# Bug 修复总结

## 修复的 Bug

### 1. 索引跟踪系统 ✓
- **问题**: `_parse_batch` 生成重复索引，导致样本权重混乱
- **修复**: 使用全局计数器生成唯一索引，建立索引-权重映射
- **影响**: 所有基于权重的选择方法（CSReL, BCSR）
- **提交**: 923dacb + f7afde4 (修复审查问题)

### 2. SimpleCNN 维度匹配 ✓
- **问题**: MNIST (28×28) 使用 CIFAR (32×32) 的特征维度
- **修复**: 添加自适应特征维度计算，根据输入尺寸动态调整
- **影响**: MNIST 数据集上的所有方法
- **提交**: f1c632c

### 3. CSReL 算法实现 ✓
- **问题**:
  - 缺少参考模型训练
  - 错误的样本选择策略
  - 权重归一化不正确
- **修复**: 完全重写 CSReL，符合原始论文算法流程
  1. 训练参考模型
  2. 计算可约损失 (Reducible Loss)
  3. 选择高 ReL 样本
  4. 按损失值分配权重
- **影响**: CSReL 方法的正确性
- **提交**: 1f00ac9 + ce6abb4 (修复审查问题)

### 4. BCSR 双层优化 ✓
- **问题**:
  - 缺少隐式梯度计算
  - 没有正则化项
  - 缺少 Simplex 投影约束
- **修复**: 实现完整的双层优化框架
  1. 内层：权重优化（Simplex 投影）
  2. 外层：样本选择（隐式梯度）
  3. 添加 L2 正则化
- **影响**: BCSR 方法的理论正确性和性能
- **提交**: d0f1bbe

### 5. 权重对齐 ✓
- **问题**: 训练时权重未正确对齐到批次中的样本
- **修复**: 使用索引-权重映射，确保每个样本获得正确的权重
- **影响**: 所有加权训练方法
- **提交**: fe02660

### 6. 损失计算 ✓
- **问题**: 返回标量损失而非每样本损失
- **修复**: 使用 `reduction='none'` 获取每样本损失
- **影响**: 基于损失的选择方法
- **提交**: fe02660

### 7. 特征提取改进 ✓
- **问题**: 回退逻辑可能返回 logits 而非特征
- **修复**: 改进特征提取策略，添加错误处理
- **影响**: 所有基于特征的方法
- **提交**: fe02660

### 8. 集成方法参数错误 ✓
- **问题**: EnsembleSelector 向 CSReLSelector 传递了不存在的 `temperature` 参数
- **修复**: 移除无效参数传递
- **影响**: ensemble 方法的可用性
- **提交**: (本次修复)

### 9. 评估方法变量名错误 ✓
- **问题**: 使用 `cls_correct` 但实际定义的是 `class_correct`
- **修复**: 统一变量命名为 `class_correct` 和 `class_total`
- **影响**: 每类准确率计算
- **提交**: (本次修复)

### 10. 实验脚本训练逻辑 ✓
- **问题**:
  - 只使用核心集数据训练，忽略新任务数据
  - 没有保存和使用核心集权重
  - 导致严重的灾难性遗忘
- **修复**:
  - 创建 `_create_combined_loader` 方法合并历史核心集和新数据
  - 保存核心集索引和权重元组
  - 在合并数据集上训练
- **影响**: 所有实验的正确性
- **提交**: (本次修复)

## 技术要点

### 索引-权重映射
```python
# 旧方法（错误）: 使用批次相对索引
indices = [0, 1, 2, ...]  # 每个批次都从0开始

# 新方法（正确）: 使用全局索引
self._global_index = 0
indices = [global_idx, global_idx+1, ...]  # 全局唯一
```

### 特征维度自适应
```python
# 旧方法（硬编码）
self.feature_dim = 64  # CIFAR 特定

# 新方法（自适应）
input_size = data.shape[-1]
self.feature_dim = input_size // 4 * input_size // 4
```

### 训练数据合并
```python
# 旧方法（错误）: 只用核心集训练
train_on(coreset_only)

# 新方法（正确）: 合并核心集和新数据
train_on(merge(coresets, new_data))
```
